How It Works

1. Server Creation

A ServerSocket is created on a specified port, listening for incoming TCP connections.

2. Connection Acceptance

The socket.accept() method blocks until a client connects, returning a Socket object.

3. Thread Spawning

A new thread is created for each client, allowing concurrent request handling.

4. Request Processing

The thread reads the HTTP request from the socket's input stream and parses the file request.

5. Response Generation

The server constructs an HTTP response with appropriate headers and sends the file content.

6. Connection Close

The socket is closed, and the thread terminates, freeing resources for the next request.
